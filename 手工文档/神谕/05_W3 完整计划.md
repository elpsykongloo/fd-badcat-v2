# 零、逐数字核验：三个疑点（W3 必须闭合）

我按你们的纪律办事，把 `grid_rollback` 的阶梯、`delta_hist` 的逐行、§3.1 的胜负明细互相对账，有三处不闭合：

**疑点二：直方图的"暴露"定义与系统真实保护逻辑是两个视角，§3.1 把它们混用了。** 逐行看：travel_19 的更正落在段间隙（3.65→4.42，间隙 0.77s **> 0.64 hold**）、housing_17 第二夹间隙 0.67s **> 0.64**，却都记 `delta_t=0`；而 ecommerce_09 间隙 0.42s **< 0.64**（hold 根本不会到期，系统视角下决策必然看到完整更正）却记 `exposed=true`。所以直方图的 exposure 是 **silero 分段视角**（"更正是否开启新段"），不是**系统视角**（"更正到达时，hold/staleness/异议窗三层保护栈处在什么状态"）。作为经验 λ（t） 的素材它没问题，但 §3.1 用它解释平台期就出现了算术裂缝：p50 1.12 + 决策 0.5 = 1.62 **> 1.5**，与"平台期左端 1.0–1.5"并不真正咬合；且阶梯 9→11→12 需要回答"δ=1.0 时抓回的是哪两夹、1.5 时补的是哪一夹、0.42 间隙那夹为何在 δ≤0.6 时会挂（hold 单层就该护住它）"。

**疑点三（最重要）：rollback 增益里混着两种效应，当前叙事只归因给了异议窗。** 证据：TACT 赢的三夹里，ecommerce_19 和 housing_25 在直方图里都是 `delta_t=0` 的"同段"夹——同段意味着两臂决策看到的最终文本相同，异议窗对它们**不可能**是胜因；可能的真因有二：（a） Phase-B prompt/管线与 blocking 的 response prompt 不同（臂间 prompt 效应）；（b） 这些夹存在**第二处修订**跨了 EoU 而直方图的单 cue 设计没记录（ecommerce_19 的"two→just one"落在 seg2，与 seg1 间隙 0.71s > hold——直方图每夹只登记一个 cue，多重修订的暴露被漏检）。干净的分解其实已经在你们的数据里：**窗效应 = TACT 臂内阶梯 +3（0.529→0.706，prompt 恒定）；臂间差 = +2 = 窗效应 +3 与 prompt/管线效应 −1 的叠加**（δ=0 的 TACT 0.529 < blocking 0.588 说明"TACT prompt 无窗"反而更差，急切脏轨迹所致）。这个分解一旦做实，图 2 从"曲线与直方图吻合"（相关性）升级为"17 夹逐夹机制归因"（因果），审稿人做同样算术时无懈可击——反过来，不做实就是被一枪打穿的靶子，n=17 上任何未归因的 +2 都扛不住质询。

# 一、总裁断：G1' 判"有条件通过"，不收缩，继续 ICLR 主线

按 03 判据字面未全过，裁断权在我，我的裁定与理由：

**P3 判"判据失效"，而非"机制失败"。** P3 写于 R9 兑现之前，其成立前提是"重叠空间存在"；实测沙箱 p50 0.315s 使重叠收益上限被压死，这是**基准延迟档的属性**，不是机制的属性。一条会惩罚基准而非机制的判据已失去诊断价值，宪法为此预留的救济就是 R9 预案——启用它（裁断 A），而不是触发收缩条款。收缩条款的立法意图是"机制展示不出价值"；实际发生的是"机制在我们预设的三个价值轴之一上结构性无空间，却在另一个轴上给出了超预期的价值"。

**P2 状态轨判"判据错设"。** 85% 是我在 03 里定的绝对阈，而你们的取证表明缺口主体是两臂共通的 ASR/实体错误（TACT state 0.570 = blocking 0.570，几乎同集合）——把机制扣在上游噪声的人质位上是错的。P2 改为**差分制**：TACT state ≥ blocking state（全量，已满足）且 rollback 状态轨 ≥ exact 轨（已满足，0.706=0.706）；85% 绝对目标移交 Revision-Bench（在我们控制 n 和录音质量的地方追求它）。

**继续 ICLR 的正面理由**：δ 扫描给出的，是比延迟胜利更好的头条——在公开基准的官方准确性指标上、在全场最难子集上，机制以 +12pt（0.706 vs 0.588）胜出，且首响与安全窗之间，完全解耦（异议窗的保护是免费的，就首响锚而言）。"惩罚脏轨迹的判分语义 + 异议窗做实帕累托点"这句话，是 ICLR 级的发现；"重叠掩蔽了延迟"，只是系统级的优化。

**诚实条件**：ICLR 的故事，现在更重地压在 W4–W5 的学习层上——因为固定 δ 的完成锚保费（+1.5s，全量 p50 2.943 vs 1.452）是真实的、必须被承认的代价，而消掉这笔保费，只能靠学习（见教义二）。R9 的兑现，因此反而是学习组件的最好动机：没有这个缺口，G2 就是个空转的门——G2（8/15）仍是真正的分叉点；W3 的职责是把 G2 建成一次公平的、仪器齐全的审判。

# 二、点名裁断（§8 加粗项 + 遗留）

**裁断 A（双档延迟评测）→ 升格为 W3 正式项，且重新定性：它不是 R9 的补救，是中心主张的另一半。** 完整主张从此是一句话两翼：官方近零延迟档下，异议窗以完成锚 +δ 的保费，换取修订鲁棒性的官方分大胜，首响零代价；现实延迟档下，首响解耦的收益，随工具时长放大，`DAG 并行` 在链式场景反超完成锚，保费占比坍缩。 

执行规范：（i） 现实档必须**可辩护地标定**——真实公开 API 延迟实测或已发表测量文献，按 κ 类分别拟合对数正态（READ 类 0.3–1s、写/预订类 1–5s 量级），标定数据与拟合参数进论文附录与可复现包；（ii） 两档都播种；（iii） **预注册预测后再跑**（防"事后拟合叙事"的审稿攻击），我的预注册版本：现实档写类 p50=3s 时，blocking 首响 ≈ 决策 1.08 + Phase 0.4 + 执行 3 + 作答 ≈ 5s 级，TACT 首响维持 ~1.14s → 比值 ~22%，**原 P3 的 ≤50% 判据在现实档达标**；单笔串行写调用的完成锚 TACT 仍 = blocking + δ（保费与档位无关，如实报）；**链式多调用场景**完成锚 TACT ≤ blocking（ActExecutor 并行 vs 串行）。跑完对表，不符处如实分析。（iv） 顺带交付：统计全量 100 里含 ≥2 个无依赖调用的场景数——若 <10，链式并行的 claim 单薄，把"链式修订场景"写进 RB 设计需求。

# 三、W3 教义：四条

**教义一：官方判分器的隐式 κ。** 

`R12 严苛分支 + 补偿必死 + 投机 READ 必死` 的现象，合起来说明一件事：官方 exact 轨把**每一个工具调用都隐式判成 IRR**——多重集 precision=1 之下，不存在"便宜的撤销"。异议窗恰好是"万物不可逆世界"里的理性策略，这解释了为什么 v0 不做任何投机，反而是官方轨最优。κ 谱系（READ⪯REV⪯COMP⪯IRR）的意义在状态轨与现实世界里才展开：补偿在那里被**计价**而不是被**处决**。这一段直接进论文的评测方法学讨论——它同时解释了 P2 预测 （iii）（κ-条件 commit 延迟单调性）为何必须**改在状态轨 + 现实档上验证**：官方轨对 κ 色盲，在色盲的裁判面前测色觉是设计错误。

**教义二：保费经济学——G2 的度量衡就在这里。** 固定 δ* 的完成锚代价是一张**全员保单**：全量 p50 2.943−1.452 ≈ 1.49 ≈ δ*，100 个场景每单都付，而理赔只发生在有跨 EoU 修订的少数场景（直方图显示官方数据里系统视角的暴露例极少）。定义**保费总额** = Σᵢ max（0， 完成ᵢ^TACT − 完成ᵢ^blocking）：固定臂 ≈ 100×δ*；oracle 臂（只在真发生修订的场景开窗）≈ 21×δ* 甚至更少；**回收上限 ≈ 79–95%**。W3 从既有 trace 直接算出 oracle 前沿（零 GPU 成本，纯重放算术），它就是 W4–W5 学习策略的靶心，G2 判据由此可以现在预注册（见 §5）。自适应策略支配固定 δ 网格 = P2 预测 （ii） 的实证形态。附带一个对 RB 设计有牙齿的推论：hazard 头的正例（意图 EoU 后修订才到达）在官方数据里只有 ~3 例——**Revision-Bench 必须把 post-EoU 修订分层过采样到 ≥50%**，否则学习组件无米下锅。

**教义三：保护栈的三层分解是图 2 的骨架。** 系统对修订的防护不是一层窗，是三层：**hold 合并**（间隙 <0.64s，更正并入同一决策输入）→ **staleness 作废**（更正到达于在途决策期间，epoch 失配弃果重判）→ **异议窗**（更正到达于 launch 后 δ 内，取消倒数 + patch pending）。台账的每一夹每一 δ 都归因到这三层之一（或 patch-after-commit 补偿、prompt/模型差异、ASR）。W1 的陈旧性协议在这个分解里第一次显出论文级价值：疑点二里 travel_19（0.77s）与 housing_17b（0.67s）这类"间隙刚过 hold"的边缘夹，大概率正是被 staleness 层接住的——若台账证实，这就是"并发化之后如何保持一致性"那一小节的活体标本。

**教义四：n=17 的统计诚实。** 0.706 vs 0.588 在 n=17 上的 Wilson 区间大面积重叠；配对视角（McNemar）下不和谐对只有 ~5 个，p≈0.375，**统计上不显著**。三重应对：（i） 台账把统计主张，转换为机制主张——T=0 确定性系统里每一次翻转都是可复现、可命名原因的事件，不是抽样噪声，这是我们相对经典 ML 论文的独特辩护，但前提是台账真的闭合；（ii） 报告一律带原始计数（12/17，不裸报 0.706）；（iii） RB 的规模由功效计算倒推：要在配对设计下让当前效应量显著，不和谐对需 ~20+，按当前翻转率推 RB 需 **60–80 个修订场景**（此前蓝图写 ~40，上调），且按修订时机分层。这三条现在写进 RB 设计文档，W4 录音安排照此执行。

# 一、屏障裁断(P0,D1 定案项):升格为显式设计,定名"提交屏障 / decision-atomic commit"

这是本周唯一不允许含糊的决定,我裁**屏障**,不裁连续时钟。三条原理,按论文可写的顺序:

**1. 它是 P1 证明在竞态区成立的前提,不是外挂。** P1 的归纳第一分支是"修订 patch 一个 pending op,无副作用"——这句话预设了 patch 抵达时 op 仍是 pending。deadline 落在 (t_eou, t_dec] 的 op 若被连续时钟提交,该区间内到达的修订只剩昂贵分支(compensate∘launch),而教义一已经证明官方判分把每个调用隐式判成 IRR——昂贵分支在官方轨等于死刑。屏障做的事情是把竞态区的修订**保在廉价分支里**,这是成本最优性的补全,不是正确性的补丁。

**2. 它是 W1 单写者纪律的对称延伸。** 决策是对 PendingSet 快照的读-改事务;快照内的 op 在事务开放期间被外部提交,就是写偏斜——决策返回的 patch 目标已蒸发。W1 把一切状态突变收归引擎协程;**提交是一次状态突变,不得发生在一个开放的决策事务底下**。我在裁断 B 里写过"决策本身也被事务化",当时是修辞,现在你们从 tx_log 里把它的机制实体挖出来了——这句话从论文的"一句话对称性"升格为语义节的正式条款。

**3. 连续时钟不省复杂性,只换一个更差的问题。** 它必须回答"patch 撞上刚提交的 op 怎么办":补偿(官方轨死)或拒绝(丢修订),二者皆劣;且它把有效救援窗耦合到推理时延抖动上——静默预算定律失效,阈值变成硬件函数,可复现性叙事(T=0/seed/A 档)被从语义层面掏空。

**engine_b 规格条款(五条,可直接写进 spec):**

1. **提交调度器是定时器驱动的**,不是"只在 EoU sweep"。回放里 sweep 只发生在 EoU 与音频结束,那是离线时间轴下的等价实现;live 引擎里用户沉默走开时 op 必须能靠定时事件提交(回放的 audio-end sweep 就是这个定时器的离线代理)。静默钟语义照搬:说话冻结倒数、patch 重启窗、δ≤0/blocking 即时提交。
2. **屏障**:决策 dispatch 时登记其快照内 pending 集 S;S 中 op 的到期提交事件在决策在途期间**延迟(不丢弃)**;DecisionDone 处理顺序 = 先应用 ops(patch/cancel、窗重启/终止),再以当前时钟执行 sweep。
3. **释放路径三条**:正常返回(上条);staleness 作废(epoch 失配)→ 在作废处理点立即 sweep;决策超时 → fail-open 立即 sweep 并记 trace(此路径在 HumDial 与全量上应零触发,封住尾延迟)。
4. **双戳记账**:每次提交记 {nominal_deadline, actual_commit},差值 = 屏障延后量,一等指标,上界应 ≤ 决策时延 p99;新增 trace 事件 `act_commit_deferred`。
5. **工程时延预算**:live 的 EoU 检测带 VAD 帧级时延 ε(silero 32ms 粒度,预算 ≤100ms),真实救援阈值 ≈ Σ静默 + ε。δ 选择准则成文:δ ≥ max(暴露阈值) + ε + 抖动余量——δ*=1.5 对 1.12 阈满足,余量 0.28s,写进规格。

**D1 验收探针(新增,硬判据):** `commit_barrier: on|off` 进 config。三翻转夹 + travel_10 对照:on → eco19@1.0 P / hou25@1.0 P / fin12b@1.5 P / travel_10 F,汇总 12/17;off + nominal 1.0s 注入 → 三夹全 F,汇总 9/17。逐位对上,语义即钉死。**这个 flag 同时就是论文的语义消融**——9/17 与 11/17 两个反事实数字不是要藏的软肋,是要亮的敏感性分析:"我们发现了这个语义分岔、形式化了它、量化了两侧",这正是把 §4A 从隐患变成资产的写法。

**后果**:grid v1 全部站得住,零重跑;δ*=1.5 头条存活且获得原则性辩护。反过来也钉死:今后任何人若翻案到连续时钟,代价是全网格重跑 + δ* 上移 + 头条重写——现在定案,永不重议。
