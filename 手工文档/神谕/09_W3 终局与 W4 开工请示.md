## 三、学习组件收缩方案（请批的核心）

蓝图 §五 Plan B = 4B 全决策头（输出整个 Phase-B 结构）。**建议收缩为"停时头"（hazard + δ 策略），Phase-B 的 ops/say 仍由 Omni + v3.1 prompt 产出**。理由：

1. **数据纪律与合成可行性的不对称**（勘正：本节初稿曾把 FDB EoU/RB 计入"可用监督"，系口径错误——**FDB/RB 是测试集，任何标签不得进训练**；hindsight-DP 在 FDB 上的产物只作 oracle 诊断上界）。训练数据只能来自：(a) **模拟器程序化合成网格**（结构/时序通道：工具链×κ×修订时刻分布按语言学统计标定，标签由 DP 算出而非人标，量不设限）；(b) **非评测真人音频**（韵律通道：HumDial 自然修订[训练许可待查，蓝图诚实差距#3] + 自录训练分片，与 RB 评测录音实例不相交；TTS 拼接增强只喂文本/时序通道——规避拼接伪影的捷径学习，前轮已分析）。在此约束下两种头完全不对称：**全决策头的 ops 语义（实体逐字、$RESULT 链、patch-vs-relaunch 判别）存在合成→真实音频的迁移缺口，缺口直接砸 exact 主表**（实测失败主体恰是实体/ASR 类）；**停时头学的是低维时序函数**（hazard 随静默/韵律/κ 的衰减），合成时序对它忠实，韵律仅小样本校准。剩余的真研究风险 = 合成场景分布→FDB 的策略迁移，由 κ 规则基线先行量出（双方同样 FDB-clean）。
2. **风险隔离**：ops 不动 ⇒ FDB exact **零回归风险**；学习收益全部落在保费/延迟侧，G2 判据干净（不会出现"准确率掉了但延迟好了"的搅浑局面）。
3. **理论贴合**：P2 的最优策略 t* = inf{t: λ(t)·C_κ ≤ c_w} 本来就只需要 λ(t) 的在线估计 + κ 查表——停时头就是 λ(t) 估计器，"理论指出学什么"的叙事一字不改。
4. **一头两用**：同一个 hazard 头同时驱动 (a) 自适应 δ（post-EoU 修订风险）与 (b) **投机派发门控**（P(EoU 确认|vad-end 前缀)，把 67.4% 浪费率压下来）。一个模型、两个消融、两处系统收益。
5. **降级路径完整**（蓝图既定）：GRPO 不收敛 → SFT-only；再降 → κ 条件规则 + hazard 特征。每级可发表。

**训练配方（蓝图 §五配方的实例化）**：

- **标注器**：hindsight-DP——模拟器确定性（音频时间轴固定 + 修订事件已知 + κ 对数正态沙箱延迟），逐 EoU 求事后最优 commit 时刻。**训练标签只在合成网格上生成；对 FDB 只作 oracle 诊断上界（83.1% 即其特例），标签不进训练**。`w3_oracle_frontier.py` 已是完成锚特例，扩展成逐 EoU 标签导出即可。
- **先行基线（防"trivial rule"攻击，SFT 前必跑，构成自适应阶梯）**：①固定 δ*（已有）→ ② κ 条件规则臂 δ(κ)（R0/R1 短、R2 长、R3 门）+ 静默钟——同时就是 P2 预测 (iii) 的第一次测量 → ③ **prompted-finality 零训练臂**：独立小调用让 Omni 听音频尾判"说完了吗"（high/med/low → δ 查表），**与 Phase-B ops 解耦**（ops 决策逐位冻结 v3.1，只动时机变量）——这是最强的免训练自适应对照，也是 audio-native 韵律敏感性的 zero-shot 测试。学习臂必须逐级击穿阶梯才有立足点；若 ②/③ 已回收大半，学习增量 claim 相应收窄——先测再训，顺序不可反。
- **模型（勘正：初稿写"4B 量级"系蓝图 Plan B 全决策头的惯性尺寸，收缩后不成立）**：停时头输出是标量（hazard/hold），不需要生成式 LM。**模型类阶梯 = 消融图本身**：v0 特征模型（逻辑回归/浅 MLP，~10⁵–10⁶ 参数，输入=静默时长、EoU 距离、κ、PendingSet 摘要、浅层词法完整性 flag）→ v1 加 0.6B 级文本编码读 ASR 前缀（语义未完判据）→ 韵律特征 W5 再加（蓝图预警的 TTS 拼接捷径学习风险：韵律通道只用真人音频，拼接增强只给文本/时序通道）。4B 仅当消融证明文本理解是瓶颈才升级。小头另有两重叙事收益：运行时 ~ms 级可逐 tick 重估 λ(t)（这正是停时问题的原生形态，Omni 逐 EoU 单发做不到），且"30B 管语义、百万参管时机"直接击破"收益来自更大模型"的攻击。
- **RL**：GRPO K=8，reward 按蓝图公式（overwait 项防保守塌缩）。单卡 96GB Blackwell：4B LoRA SFT + 模拟器 rollout 完全可行，Plan A（MoE-LoRA 30B）维持放弃。

**G2'/G2 判据（请神谕确认或修订）**：学习臂满足——(i) 帕累托支配**全部固定 δ 点 + κ 规则臂 + prompted-finality 臂**；(ii) 保费回收 ≥ 51.5s/100 场景（= oracle 上限的 62%）；(iii) exact 对 δ* 固定臂差 ≥ −1pt；(iv) κ 单调性成立。四条中 (i)(iii) 必须，(ii) 未达但 >0 时按实报并降低 claim 强度。加分项：投机门控把浪费率从 67.4% 压到 ≤35% 且首响 p50 不高于 0.70。若 (i) 里学习臂只平不超 prompted 臂：G2 判负，叙事回落为"停时规则的 zero-shot 可实现性"，投稿树走 ICASSP 分支——这正是 8/15 决策日的既定用途。
