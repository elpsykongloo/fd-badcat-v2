# 二、W2 教义:Q1 实锤如何反转为全文杠杆

你的审计揭示的判分语义,值得把推论一次性推到底,因为它**重写了策略空间**:

**推论一:官方 exact 轨下,任何多余调用 = 场景死刑,包括 READ。** 补偿序列必死你已实锤;更狠的是投机 READ 也死——若修订发生,`search(Paris)+search(Berlin)` 的多重集 precision 即破。所以**官方轨最优策略族里根本没有"抢跑"这个动作**,一切投机执行的价值只存在于状态轨。这直接解释了遗留之谜:**async 0.71 < blocking 0.73 的最大嫌疑就是旧 decider 的急切调用被严格判分处决**(W2 Day 5 用逐场景失败差分验证,这是义务不是选做)。

**推论二:官方轨的延迟收益必须来自"不增加调用的重叠"。** blocking 的时间线是 `听完→LLM决策→执行(干等)→LLM作答→TTS` 的纯串行;TACT 在**调用集完全相同**的前提下重排:决策即发射 + 执行期间 ack/叙述占据首响锚点 + 结果到达即作答。首响锚(Q5:输入结束→首非静音)被一句短 ack 直接压到"判定+短句合成"的量级;任务完成锚吃掉 `min(执行时长, 作答准备时长)` 的重叠量。**这就是三难叙事在 v3 判分语义下的精确形态:blocking 准确但串行慢,旧 async 快但轨迹脏,TACT = 净轨迹 + 重叠。**

**推论三:跨轮修订场景的官方最优解,恰好是蓝图 P2 的 grounding 门——而固定时间轴逼出了它的正确形态。** 21 个场景的 expected 只含最终意图 ⇒ 官方期望系统在修订到来前**没有**发出首次调用 ⇒ 写类调用必须过一道提交门。但 Q2 说用户音频是固定时间轴、非反应式——阻塞式的确认问句会与脚本脱钩(用户不会回答你没被脚本预料的问题)。解法是把确认从"话轮依赖"改为"计时门":**声明式提交 + 异议窗(announce-and-commit)**——系统播报"正在为您预订柏林 3 点的航班"(此播报同时充当 ack,覆盖首响锚),随后开一个 δ 秒的异议窗:窗内到达的修订 → patch pending,零副作用;窗竭 → commit。固定时间轴下,脚本里的更正语音("啊等等,改成波士顿")天然落进窗内 → 单次最终调用 → 官方 PASS;真实用户场景下,这就是人类客服的自然行为("订3点那班了哈,有问题您说")。**δ 就是 P2 的停时变量,而官方 pass rate 就是 C_κ 的操作化**——你在 §4 里"机会面"那段的直觉,推到底就是这个。

**推论四:经验 hazard 是免费午餐。** 21 个场景的音频时间轴里,"触发意图话语结束 → 更正到达"的间隔分布**就是 λ(t) 的直接经验估计**——W2 Day 2 把它量出来(直方图),它同时是:δ 扫描范围的设定依据、P2 膝点预测的 x 轴刻度、W5 hazard 头的第一份标注数据。理论闭环的第一圈在 W2 就能转起来,而且转在**官方基准的官方分**上。

**推论五:双轨判分的角色分工从此清晰。** 官方 exact 轨 = 帕累托主战场(净轨迹 + 低延迟);judge 轨 = 语义宽容度参照(**W2 Day 1 补一项审计:judge 轨是否也执行多重集 precision=1**——若 judge 轨宽恕多余调用,它就是补偿机制的半个官方庇护所,叙事再加一层);状态轨(你要建的终态判分器)= 事务机制的价值显影剂,eager+compensate 在这里以"更低延迟、同等终态正确"的形式被正确记账。三轨并报,各司其职。

一句话教义:**W2 的 TACT v0 不做任何投机,只做"异议窗提交门 + 无额外调用的重叠"——用最保守的机制拿下官方轨帕累托,把激进机制(投机/补偿)留给状态轨与 Revision-Bench 去展示上限。**
