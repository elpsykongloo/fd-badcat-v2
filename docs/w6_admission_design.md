## 一、问题定义（两条已量化的准入缺口）

固定 δ 窗口 + 提交屏障解决了"**何时**允许修改"（batch-1 判定：屏障支持、
δ 单调、无屏障净负）。test-897 两批留下的真实缺口都在"**谁**与**改什么**"：

- **WHO 轴（L10，批一 §5.3）**：窗口对任何声音开放。TACT 第三方命令拒绝率
  .500(A)/.154(B) vs blocking .929/.692 = 开放窗的安全代价；良性本人修订穿透
  .615(A) 是要保住的收益。蓝图 §4.6 的 SV 门控即此，W1 起一直空缺。
- **WHICH 轴（L7 反窗口夹，批二 §7.3 勘误版）**：6/6 夹形状一致——修订语义
  指向第二步槽位（country/item_id），决策器却 patch 进仍开窗的第一步 search
  （逐夹 ops 有 `patch op_id=2` 直接证据）；blocking 单发合并同样错 ⇒ 归属
  错误是模型级缺陷、**开着的窗口是错误吸附面**（放大器）。commit-now 移除
  吸附面故"更快=更对"，但代价是丢掉 16 个归属正确的真救援夹——分开这两类
  只能靠归属判断，不能靠任何窗口长度政策（含已判死的学习头）。

结构事实（2026-07-16 由 test 归档行核验）：多声音 episode **全部且只在 L10**
（26/897）⇒ WHO 门在其余 871 夹结构性不触发；misattribution 夹集中 L7
（test 内至少 6 夹有直接 patch 证据）。

## 二、机制设计

### 2.1 SV 门（WHO）——决策级准入

每个决策由某个语音段的 EoU 触发（runner：`segs[i]`）。门 = 该触发段的说话人
验证不通过时，**丢弃本决策的全部 ops**（launch/patch/cancel），say/对话路径
不受影响；trace 记 `sv_rejected`，被丢弃 ops 记录在案（审计轨）。两行：

- **SV-oracle**：用构建期 voice 标签（arm A cues / arm B piece voice）做完美
  归属 = 机制价值上界。**构造保证**：单声音 episode 逐位不变（可 assert）。
- **SV-real**：真实嵌入器。enrollment = episode 第一个用户段；每触发段取
  余弦相似度，τ 阈值判决。嵌入器选型 = CAM++/ERes2Net（modelscope，与
  SenseVoice/FSMN-VAD 同生态、本地确定性）或 SpeechBrain ECAPA，Phase-1 听检
  后定一个；τ 在 **RB-dev 上冻结**（103 夹含 L10 dev 夹），test 前入 §6。

已声明难点（如实入册，不美化）：arm A 的 bystander 段按生命周期偏移混音，
可能与用户语音**时间重叠** ⇒ 触发段嵌入是混合声；TTS 声比真人干净 ⇒ 分离
偏易，金子集（144 项，真人）是后续效度锚。

### 2.2 归属门（WHICH）——修订绑定规则

失败机制在决策器内部（语义绑定），故主形态 = **RB prompt 增量规则**（决策
消息内、不加新调用不加延迟）：修订必须绑定到"其值语义所属字段"的 op/动作；
若指向尚未 launch 的动作的字段，**带值 launch 该动作**，不得 patch 语义不符
的 pending op。措辞在 dev 上**最多两轮**迭代（沿 FDB 五靶纪律：不中不得
无限调词），定稿措辞冻结入 §6。工程约束：规则作为独立常量叠加（与
`PROMPT_V31_ADDENDUM` 同构），provider 约定 `*_attr`；不动 v3.1 本体 ⇒
非 attr 臂缓存键不变。

### 2.3 组合臂

SV-real + 归属门同装（`*_svattr`）：验证两门正交性（预测：增益近似相加，
因作用夹集不相交——L10 vs L7）。

## 四、预注册判读（冻结；数值带 dev 定稿后只许收紧不许放宽）

- **P1（SV 主判读，L10-A n=14/13）**：SV-oracle 对抗拒绝 .500→**14/14**（构造
  上界），SV-real ≥ **13/14（≥.929，追平 blocking）**；良性穿透 **不降**
  （≥8/13）；非 L10 层通过集：oracle 逐位同（assert），real 允许 ≤1 夹误伤
  预算（超预算 = 门失败，报负结果）。
- **P2（SV，L10-B）**：拒绝 .154→≥.692（追平 blocking）；armB 良性 .333 不降。
- **P3（归属门，L7）**：收复 6 个 misattribution 夹中 **≥4**；L3/L4 的 15 个
  window-rescue 夹 **0 损失**；全量 exact **≥ .2662**（不接受零和）。
- **P4（组合）**：svattr ≈ svreal 增益 + attr 增益（偏差 ≤1 夹）。
- 判读纪律照旧：n<30 的格描述性报告；配对计数 + 精确二项。

**Kill 判据**：K1 = SV-real 在 dev 单声音夹上误拒 >2（嵌入器/阈值不可用 ⇒
只发 SV-oracle 行 + 感知差距分析，WHO 门主张降级为机制上界）；K2 = 归属规则
两轮 dev 后 L7 收复 <2 或 L3/L4 出现任何损失（词条零和 ⇒ 按五靶纪律封盘，
负结果 + dev 收据入册）；K3 = runner v1.2 oracle 平价复放不过（工程门，修完
才许进 dev）。

## 七、基准合理性分析的归档位（回应"共同失败=任务不可能？"）

零 GPU、由 test 归档行重算，进论文 benchmark 节而非新实验：541 夹中 oracle
可过 420（77.6%）；TACT/blocking 共同失败 382 夹中 **270（70.7%）oracle 可过**
= 可完成但内容层失败（听错槽位/漏工具/旧值），真·机制天花板只有 112
（20.7%，L5/L6 设计保留的区分余量）；可完成子集上 TACT/blocking = .338/.264
（差分不变）。外锚：τ-voice 最强商用系统任务完成率 ~38%。结论：**绝对分低的
主体是单一 30B 底座的内容决策质量，不是任务不可能，也不是机制失效**；抬绝对
分的合法杠杆 = 更强底座（模型动物园，新系统版本单发）与内容侧规范化（已裁定
非本文命题）。设计教训如实记：内容轴难度（双语 verbatim/规范格式/引用语法）
与本基准命题（修订时机）正交，v2.3 若做应把难度质量集中到时机轴——本文不重建
（test 已耗尽 + 单发叙事完整 + 相对主张不受影响）。
