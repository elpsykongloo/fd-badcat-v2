## §0 一句话

学习一个**话语完成度（finality）门**，在 vad-end 时刻决定是否投机派发决策推理：把投机派发的废单率 **67.4% → ≤35%**（W3-09 号文既定加分判据转正为主门），同时首响地板 0.640s 保全、**决策内容逐位不变（结构保证）**。它是 W4 停时头负结果的建设性对偶——W4/V3 证明**意图稳定性不可学**（LOO 43.9%、韵律盲、探针双 FAIL），而**话语完成度恰是探针证明可学的那一侧**（HumDial 结构族 S AUC 0.9841；Omni 读数 break_mid→unfinished 85.3%）。双分离 = 论文分析章的构成性主张，本项目为其补上"学习用在信号在的地方"的正结果位。

## §1 背景与靶（全部为已归档实测）

- 投机派发（W3 D6，live full engine）：**604 派发 / 197 确认 / 407 作废 = 废单率 67.4% ≈ 2.8× LLM 调用**；收益 = 首响 p50 1.218 → **0.640 = 物理地板精确命中**（P3 比 0.419）；exact 0.630（ε 带 −1，感知敏感类，非机制回归）。
- 09 号文加分判据（原文）："投机门控把浪费率从 67.4% 压到 ≤35% 且首响 p50 不高于 0.70"。本预注册将其转正，并收紧首响条款（§5）。
- 机制事实（关键，决定了本项目的风险形状）：**门控只改"是否预计算"，不改 EoU 确认时刻的决策输入**。被门挡掉且后来确认的 EoU 走非投机路径 = 同一快照、同一决策、更晚到达。⇒ **决策内容逐位不变是构造性质，不是评测目标**（§5 A1 为 assert 而非门）。项目的全部赌注在效率-延迟前沿，不在准确率。

## §2 任务定义（冻结）

- **预测对象**：在引擎 vad-end 事件时刻，预测 `y = 1[该 vad-end 起 0.64s 内无语音恢复]`（=hold 将到期、派发将被确认）。y=0 ⇔ 派发将作废。
- **部署动作**：`P̂(y=1) ≥ θ` → 照常投机派发；否则不派发（该 EoU 若确认则回退非投机路径）。单调阈值策略，无第二段。
- **记账口径**（冻结）：`waste = 被作废派发 / 总派发`；`missed = 被门挡掉且实际确认的 EoU / 全部确认 EoU`；首响投影 `first = max(0.64, infer)`（派发且确认）/ `0.64 + infer`（missed）。

## §3 数据与合规（冻结）

- **训练/校准数据 = HumDial HD-Track2 train 独占**（9,988 样本 / 106.9h / 双通道；许可已获，含训练用途）。两通道各自作为"被听说话人"独立抽取事件（申报于 Phase-1 报告）。
- **标签提取器必须与引擎 VAD 逐参数一致**：用引擎同款 silero VADIterator 配置（512 样本/32ms 粒度、引擎阈值）重扫音轨，**不得复用 w4v3 普查的 400ms 分析口径**——那一口径下 <400ms 的间隙根本不产生 vad-end 事件，而 [0, 0.64) 恰是本任务的负类主体。此条为 Phase-1 的第一交付物。
- **FDB = 测试集**：训练、特征选择、θ 选点全程不得接触任何 FDB 信号；FDB 记账为**单发**（§7 K3）。RB 同为测试集（建成后）。
- **合规红线**：产物零 HumDial 转写原文（`assert_no_text()` 强制）；只落时长/计数/哈希/标签。

## §4 特征与模型政策（冻结）

- **F_time（默认族）**：vad-end 时刻引擎可得的音频钟观测——当前话语时长、话语内 vad 段数、前 3 个间隙时长、距轮首时间、近 5s 语音/静默占比、vad-end 在轮内的序位。
- **F_text（可用性门控族）**：ASR 前缀结构统计（token 数、末 token 词类〔复用 w4v3 词表〕、完整性 flag）。**进入条件 = Phase-1 实测 ASR 前缀在 vad-end 时刻的可用性时滞**：若 live 路径上前缀到达晚于派发时点（p50 时滞 > 50ms），该族整体排除（离线可算≠部署可用）。
- **F_pros（韵律族）**：**默认排除**——w4v3 探针已证其对 finality 无增量（T1/T2/X 双 FAIL）。仅当 F_time(+F_text) 在 Phase-1 探针上经济可行域为空、且新目标（0.64s 恢复预测 ≠ 停顿-终止判别）的专项探针显示显著增量时，方可另立附录预注册引入。
- **模型阶梯**：LR 主臂 + MLP(h=16) 消融（沿 v2 惯例）；容量升级判据沿 `w4_ladder_design` §12.4 双条件纪律（迁移逼近域内上限仍不过门 ∧ 探针证明上限可抬）——资源不设限≠归因可以失效。
- **分组纪律**：按 dialogue 分组的 train/val 切分与 OOF 探针（防同对话泄漏，沿 w4v3 group_fold）。

## §5 门与判据（冻结；数值锚 = FDB live 档案 604/197/407）

| # | 条款 | 阈值 | 性质 |
|---|---|---|---|
| G1 | 废单率（FDB 全量账面重放） | **≤35%**（基线 67.4%） | 主门 |
| G2a | missed（确认 EoU 被挡比例） | **≤15%** | 主门 |
| G2b | 首响投影 p50 | **= 0.640 不动**（G2a⇒结构保证，复核位） | 主门 |
| G2c | 首响投影 p90 | ≤ spec-on 档案 p90 + 0.15s | 主门 |
| A1 | 决策内容 | 工具调用/pass **逐位一致**（构造保证） | assert |
| A2 | HumDial 冒烟 | 三判配对 Δ≥−1（照例；预期恒等） | 护栏 |
| — | live 实测确认批（GPU 日） | waste/first 与账面投影一致性 ±5pt/±0.05s | 收口 |

**AND 判据 = G1 ∧ G2a–c**。等价 ROC 表述：FDB 确认基率 32.6% 下，G1∧G2a ⟺ 派发精度 ≥0.65 @ 召回 ≥0.85。

## §6 θ 选点规则（冻结）

θ 在 **HumDial val（分组切分）** 上按冻结规则选定：`θ* = argmax precision(θ) s.t. recall(θ) ≥ 0.85`；若可行集为空取 recall 最高点并如实报。θ 一经选定即写入 stophead JSON（provenance 含 config 哈希），FDB 账面重放**单发**。HumDial↔FDB 的基率/分布差为申报风险（同 v2 之例），**不得用 FDB 结果回调 θ**。

## §7 kill 判据（冻结）

- **K1（标签量）**：引擎口径 vad-end 事件（双通道合计）< 20,000 ⇒ 取消（预期 ≫ 此值，护栏位）。
- **K2（域内可行域空）**：HumDial OOF 探针上 `precision@recall=0.85 < 0.50` ⇒ 取消，零训练浪费；负结果（"0.64s 恢复在该特征类不可判"）併入双分离分析章，如实反向修正"finality 可学"的适用边界。
- **K3（域移负结果）**：FDB 单发账面若 waste >55% ∨ missed >25% ⇒ 记负结果收口，**不得重调重跑**；HumDial 域内数字仍可作论文素材（部署域外推失败=诚实边界）。

## §8 Phase-1 清单（零训练；完成后回填 §9 数值冻结）

1. **引擎口径标签普查**：`scripts/w5sg_census.py`——引擎同参 VADIterator 重扫 HumDial train 双通道；输出事件表（时刻/前后间隙/话语时长/序位；无文本），统计 y 基率、间隙分布、与 FDB live 604 事件分布的形状对照。
2. **ASR 可用性审计**：vad-end 时刻 ASR 前缀时滞分布（决定 F_text 进出）。
3. **OOF 探针**：F_time（±F_text）对 y 的分组 5 折 AUC 与 precision@recall 曲线；K2 裁定。
4. **数值冻结（§9 回填）**：确认/修订 G 表数值锚（若 HumDial 基率与 FDB 差异过大需换算式，换算式此时冻结）+ 点预测 P-SG1..4 锁定（waste/missed/first 投影/HumDial 域内 AUC）。

## §9.1 Phase-1 判决（2026-07-15，用户实跑，提交 6f41c85）——**K2 FAIL，v0 按 §7 取消**

- **census**：9,988 samples / **102,194 vad-end 事件**（K1 PASS，5× 于 20k 门）；确认基率 **0.3961**（FDB live 32.6% 的同数量级邻域）；gap 分位 zh p10/25/50/75/90 = 0.1/0.2/0.3/2.5/2.8s、en = 0.1/0.2/0.3/0.8/2.7s——**负类主体是 ≤0.3s 微停顿**；ASR 可用性审计：投影时滞 p50 **52.5ms > 50ms 冻结门 ⇒ F_text 排除**（以 2.5ms 之差；按冻结规则执行，不弯折）。数据修复记录：英文样本误标 zh 的字段映射修复后全链重跑，K2 数值不变。
- **probe（F_time 6 维，分组 OOF）**：AUC **0.6425**；precision@recall0.85 = **0.4228**（仅高出基率 +2.7pt）**< 0.50 ⇒ K2 FAIL**。
- **判决（冻结规则机械执行）**：v0 取消，零训练浪费、FDB 零接触。负结果定性：**"本次静默会持续多久"在纯时序历史上不可判**——话语时长/前序间隙/局部密度对 0.64s 恢复预测只携带弱信号（AUC 0.64）。与 w4v3 探针（内容结构族对"停顿-终止"判别近饱和 AUC 0.984）合成更锋利的图景：**finality 信号是内容承载的（content-borne），而部署时点（vad-end 即刻）的可观测集里只有时序——内容恰好被 ASR 时滞墙挡在外面（52.5ms）**。双分离主张措辞相应收窄："finality 可学"限定为离线内容探针口径；在线 timing-only 版不可达经济工作点。
- 〔后继已立项：见 §12 SG v1 预注册（2026-07-16，用户裁定"完整彻底立 v1"）。〕
- **后继路径（v0 判决时的原始记载）**：地板余量算术给出干净的候选假设——首响地板对派发延迟的容忍度 = 0.64 − infer p50 0.561 = **79ms ≥ ASR 时滞 p50 52.5ms**：**"派发等 ASR 前缀"的架构改造在 p50 上结构性保住地板**（first = max(0.64, lag+infer)，lag+infer p50 ≈ 0.614 < 0.64）。任何此类 SG v1（ASR 同步门控 + F_text）必须新预注册：新 Phase-1 = 对 census 102k 事件补 ASR 结构特征（SenseVoice 全量 ≈2.3h，RTF 0.021×106.9h）+ 重跑分组 OOF 探针 + **同门 K2'（precision@recall0.85 ≥ 0.50，不得放宽）**；合规：ASR 转写缓存必须落在仓外（如 /root/autodl-tmp/w5sg_asr_cache），入仓产物仍数字 only。若 K2' 亦败，SG 线以双探针负结果永久收口（timing 与 ASR-content 双轨均不可达），分析章素材完整。

## §10 相关工作定位（写作素材，非判据）

- **SHANKS (2510.06917)**：边听边想、用户未说完即完成 56.9% 工具调用——占据"提前执行"极大化。本项目目标不同：在**修订安全不变量**（R0/R1 inert、EoU 确认前无副作用、决策内容不变）之上优化既有投机机制的**浪费经济学**——SHANKS 最大化提前量，我们在给定提前量（地板 0.64）下最小化计算废单。
- **AsyncLM (2412.07017)**：文本域异步/可中断函数调用（BFCL 1.6–5.4×）——异步范式先驱，无语音、无修订语义、无浪费记账。
- **MoshiRAG (2604.12928)**：keyword-delay 隐藏异步检索——利用等待期的固定启发；我们是学习门控 + 显式经济学。
- **预测式 ASR 投机执行**：投机+推迟副作用的先例；差异同上（决策级门控 + 废单率主张 + 全双工修订语境）。
- 与 W4 停时头的关系 = 双分离两翼：意图稳定性（不可学，负结果+解剖）vs 话语完成度（可学，本项目正结果位）。**若 K2/K3 触发，双分离主张须如实降级**——这是本预注册对论文叙事的可证伪暴露，特此声明。

## §11 风险

1. HumDial（对话域）→ FDB（任务指令域）停顿分布移位：已由 C1 世界锚实证（真实停顿更长）；方向利好（FDB 短停顿更多=负类更易判）不作假设，K3 兜底。
2. 引擎 VAD 参数与 HumDial 录音信道差（近讲 vs 远场）：Phase-1 分布对照条目专查。
3. F_text 可用性失败 ⇒ 只剩 F_time：探针裁定，可行域空则 K2 干净退出。
4. live 实测与账面投影漂移：workers 口径墙钟非权威（已知），收口批走串行 GPU 日。

## §12 SG v1 —— ASR 同步门控（新预注册，Phase-0 结构冻结 2026-07-16）

> 立项依据：用户 2026-07-16 裁定。v0 判决（§9.1）保持不变；v1 是**新预注册**，不是 v0 重跑。

### §12.1 与 v0 的唯一结构差异（冻结）

**派发时点从 vad-end 即刻改为 vad-end + 当段 ASR 到达**（实测投影 p50 52.5ms）。地板余量算术：`first = max(0.64, lag+infer)`，lag p50 0.0525 + infer p50 0.561 = 0.614 < 0.64 ⇒ **p50 首响地板结构性保住**；代价在 infer 慢尾（p90 侧 +lag 线性）。特征族 = **F_time（6 维，同 v0）+ F_text_asr**：当前话语 ASR 前缀的结构统计——n_chars、n_tokens + w4v3 词表族 `t1_features`（尾 token 词类；与 w4v3 探针同源实现）。F_pros 仍默认排除（w4v3 双 FAIL）。

### §12.2 门与判据（冻结；与 v0 同门，不放宽）

- **K2'**（Phase-1 探针，训练前）：HumDial 分组 OOF `precision@recall0.85 ≥ 0.50`，否则取消——**SG 线以双探针负结果永久收口，无 v2**。
- G1/G2a/G2b/G2c 与 §5 逐字不变（G2b 的 p50==0.640 依 §12.1 算术仍可达）。θ 选点规则 §6 不变（HumDial val，FDB 单发，K3 不变）。
- 诚实预测（Phase-1 前锁定）：w4v3 内容探针 0.984 要打两重折扣（干净标注文本→带噪 ASR；配对判别→全事件打分）。**赌点 = 折扣后仍 ≥0.50**；预期带：OOF AUC ∈ [0.72, 0.90]，prec@rec85 ∈ [0.45, 0.75]——带下沿意味着可能恰好双输，如实接受。

### §12.3 合规（冻结）

ASR 转写缓存**仓外落盘**（默认 `/root/autodl-tmp/w5sg_asr_cache/`，逐段 JSON，键=sha256(wav:seg)）；入仓产物（events_v1.jsonl / meta / 报告）**数字 only**（`assert_no_text` 强制）。HumDial 许可与红线同 §3。

### §12.5 Phase-1 判决（2026-07-16，实跑）——**K2' FAIL，SG 线永久收口**

- **提取与同源核验**：SenseVoice 实际资产为仓内忽略目录下的
  model.int8.onnx（提取器补齐与 backend 同款的 int8 优先、full fallback，
  不改特征/标签定义）；6 样本串行与 4 worker 对照的 90 行事件逐字节同
  SHA-256。全量以 64 worker 跑完：**9,988 samples / 102,194 events**；
  与 v0 census 的标签、gap、语言、group 与前 6 维时序特征逐行
  **0 mismatch**，新增后每行固定 15 维有限数值。
- **合规与完整性**：入仓 events_v1.jsonl SHA-256 =
  79b8c3ed84a12f7766c10cd11ce49d99ab1c55ceda74d42e26f693427cb6f52d，
  数字-only schema 检查 0 例违规；仓外
  /root/autodl-tmp/w5sg_asr_cache/ 共 102,194 个可解析段缓存，0 坏文件、
  26 个空转写。原始转写不入仓。
